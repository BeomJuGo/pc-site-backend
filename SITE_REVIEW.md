## ❌ 발견된 문제점

### 1. **AI 추천 페이지에 용도(purpose) 선택 UI 누락** ⚠️ **치명적**

**문제:**
- AI 추천 페이지(`/ai-recommend`)에 예산 입력 필드만 있고, **용도 선택 UI가 없습니다**
- 스냅샷 상에 용도 선택 드롭다운, 라디오 버튼, 또는 다른 입력 필드가 보이지 않음
- 백엔드에서는 `purpose` 파라미터가 필수인데, 프론트엔드에서 전송할 수 없음

**현재 상태:**
- 예산 입력: ✅ 있음 (spinbutton, 기본값 1,500,000원)
- 용도 선택: ❌ 없음
- "추천 받기" 버튼: ✅ 있음

**영향:**
- 사용자가 AI 추천을 받을 수 없음
- 백엔드 API 호출 시 `purpose`가 없어서 검증 실패 가능

**해결 방안:**
```jsx
// 프론트엔드에 용도 선택 UI 추가 필요
<select value={purpose} onChange={...}>
  <option value="게임용">게임용</option>
  <option value="작업용">작업용</option>
  <option value="사무용">사무용</option>
  <option value="가성비">가성비</option>
</select>
```

### 2. **API 연결 오류** ⚠️ **치명적**

**문제:**
- CPU 카테고리 페이지에서 `ERR_CONNECTION_REFUSED` 오류 발생
- 백엔드 서버가 `http://localhost:10000`에서 실행되지 않음

**에러 로그:**
```
[ERROR] Failed to load resource: net::ERR_CONNECTION_REFUSED @ http://localhost:10000/api/parts?cate...
[ERROR] [fetchParts] error: TypeError: Failed to fetch
```

**영향:**
- 모든 카테고리 페이지에서 데이터를 불러올 수 없음
- CPU 카테고리: "총 0개" 표시
- 부품 목록이 표시되지 않음

**해결 방안:**
1. 백엔드 서버가 실행 중인지 확인
2. 포트 번호 확인 (프론트엔드가 10000을 기대하는데 백엔드는 다른 포트일 수 있음)
3. CORS 설정 확인

---

### 3. **로딩 상태 처리 개선 필요** 💡

**문제:**
- CPU 카테고리 페이지에서 "로딩 중..." 메시지가 표시되지만, 실제로는 API 오류로 인해 로딩이 완료되지 않음
- 타임아웃 처리가 없어서 무한 로딩 상태가 될 수 있음

**해결 방안:**
- 로딩 타임아웃 추가 (예: 30초)
- 에러 발생 시 사용자에게 명확한 메시지 표시
- 재시도 버튼 제공

---

### 4. **에러 메시지 표시 개선 필요** 💡

**문제:**
- API 연결 실패 시 사용자에게 명확한 에러 메시지가 표시되지 않음
- 콘솔에만 에러가 로그되고, UI에는 반영되지 않음

**해결 방안:**
- 에러 바너 또는 토스트 메시지로 사용자에게 알림
- "서버 연결 실패" 같은 명확한 메시지 표시
